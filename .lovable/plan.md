## Що робимо з блоком "Послуги" (Варіант B)

Працюємо тільки з файлом `src/components/home/ServicesSection.tsx`. Інші файли не чіпаємо. Якщо результат не сподобається — кнопка revert під моїм повідомленням поверне все назад.

### Нова структура

Замість 3 однакових карток + великої CTA знизу — **2 контрастні картки-сценарії** (це і є реальні шляхи замовлення експертизи):

```text
┌──────────────────────────────┐  ┌──────────────────────────────┐
│  [icon Gavel]                │  │  [icon FileSignature]        │
│  За ухвалою суду             │  │  За заявою сторони           │
│  Для суддів та слідчих       │  │  Для адвокатів і громадян    │
│                              │  │                              │
│  ✓ Прийом ухвали/постанови   │  │  ✓ Безкоштовна консультація  │
│  ✓ Виїзд експерта на об'єкт  │  │  ✓ Договір і строки наперед  │
│  ✓ Висновок як доказ у справі│  │  ✓ Висновок експерта         │
│                              │  │                              │
│  [Деталі →]                  │  │  [Замовити експертизу →]     │
└──────────────────────────────┘  └──────────────────────────────┘

       Що входить у вартість →   (тонке текстове посилання під сіткою)
```

### Зміни по пунктах

1. **Шапка секції** — залишаємо `Послуги` + заголовок, але скорочуємо підводку до **одного речення**: "Два шляхи отримати експертний висновок — оберіть зручний." Прибираємо другий довгий параграф (він дублює About).

2. **Дві картки замість трьох:**
   - Ліва: **"За ухвалою суду"** — для судів/слідчих. Іконка `Gavel` (lucide). Стиль: світла з border. CTA: `Деталі →` → `/posluhy/ekspertyza-za-ukhvaloiu-sudu`.
   - Права: **"За заявою сторони"** — для адвокатів і фізосіб. Іконка `FileSignature`. Стиль: акцентний (фон `bg-brand-blue`/темний, білий текст) — щоб виділити основний комерційний сценарій. CTA: `Замовити експертизу →` → `/posluhy/ekspertne-doslidzhennia-za-zaiavoiu`.
   - В кожній картці — підзаголовок "для кого" + 3 буллети по суті процесу (з іконкою `Check`).
   - Картки одного розміру, `grid-cols-1 md:grid-cols-2 gap-6`.

3. **Третя картка → дискретне посилання.** "Що входить у вартість" виносимо під сітку як текстове посилання з іконкою (`Дивитись, що входить у вартість →`) → `/posluhy/shcho-vkhodyt-u-vartist`. Так не дублюємо рівень важливості.

4. **Прибираємо дублюючу велику кнопку "Замовити консультацію"** знизу секції — вона вже є в Hero, у floating-кнопці і в Footer. Менше шуму, чіткіший фокус на двох картках.

5. **Фон секції** — змінюємо `bg-gray-50` → `bg-white`, щоб не зливалось візуально з сусідніми секціями (Hero/ExpertiseCarousel вже мають свої фони).

### Технічні деталі

- Імпорти оновити: замість `FileText, Handshake, DollarSign` → `Gavel, FileSignature, Check, ArrowRight`.
- `services` масив залишається з 2 елементами (структура та сама — `id, icon, title, audience, bullets[], slug, ctaLabel, variant: 'light' | 'accent'`).
- Темну/акцентну картку оформити через умовні класи (`variant === 'accent' ? 'bg-brand-blue text-white' : 'bg-white border'`).
- Іконки буллетів `Check` — кольору `brand-blue` на світлій картці, `white/80` на темній.
- Hover: легкий `translate-y-[-2px]` + посилення тіні (як зараз).
- Адаптив: на мобільних дві картки одна під одною (`grid-cols-1`).

### Що НЕ зачіпаємо

- Маршрути, slugs (`ekspertyza-za-ukhvaloiu-sudu`, `ekspertne-doslidzhennia-za-zaiavoiu`, `shcho-vkhodyt-u-vartist`) — лишаються ті самі.
- Інші секції головної (`Hero`, `ExpertiseCarousel`, `VideoRecordings`, `FaqSection`, `PartnersSection`).
- SEO/structured data, Footer, Navbar.
